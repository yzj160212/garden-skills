# 十条不可妥协的原则

"视频感的网页"和"PPT 感的网页"的分水岭。

- **结构原则**（1-5）由 `templates/` 已经实现，agent 写章节时受益
- **内容原则**（6-10）每写一步都要主动检查 —— 大多数"AI 味浓"翻车的地方

> 这是 SKILL 的"宪法"。所有详细推论（CHAPTER-CRAFT 视觉法则、
> OUTLINE-FORMAT 心法）都派生自这里。其它文件**只引用条目编号**，
> **不重复展开** —— 看到"参见原则 7"就回这里查。

---

## 1. 16:9 固定舞台，按视口缩放

内容固定 1920×1080，`transform: scale()` 缩到视口，外围 letterbox。
**没有响应式断点**。

## 2. 一个全局 step 计数器驱动所有变化

点击舞台空白 = `step++`，键盘 ←/→ 也行。章节 = `step` 的纯函数，
**没有动画定时器，没有命令式状态**。

## 3. 每一步独占整个屏幕

```tsx
if (step === N) return <FullSceneN />
```

`key={step}` 重挂动画。**不要** `step >= N && <ExtraThing />` 往下堆。

## 4. 口播节拍 === step（一一对应）

一节拍 = 一 step = 一个聚焦想法。**时长不由 agent 控制 —— 录制时讲者
按口播节奏走，动画跑完就停在原地等下一次点击**。所以动画要做到"出
现得干净利落、停下来不抢戏"，而不是去精算"刚好 X 秒讲完"。具体见
[`CHAPTER-CRAFT.md`](CHAPTER-CRAFT.md) Part 2 时长参考。

## 5. 隐藏的边角控件

进度条 / 翻页器 / debug 计数器默认 `opacity: 0`，`:hover` 才显示。
屏幕录制器看不到 chrome。

---

## 6. 舞台上不要 header / footer / 页码 / 品牌条 / 面包屑

把网页演示做成 PPT 的头号原因。章节标题应该是**该章第一步 hero
动画的内容**，下一步就消失。**口播是位置信息，进度条是兜底**。

---

## 7. 内容驱动动画 —— 找出"这一步要演什么"，找不到才用入场动画

视频感 ≠ "屏幕上一直有东西在动"。**装饰性的伪动画**（无差别给每步
挂 ken burns / 字符光晕呼吸 / dot pulse / 流光 sweep / 缓慢扫描线 /
背景粒子漂移……）**不会让网页变成视频，只会让画面变冗、抢戏、
显得 AI 味重**。

**真正的"动画"是把这一步的内容动作演给观众看。** 写 step 时第一个
问题永远是 ——

> **这一步的内容里，有什么本身就是"动作 / 变化 / 关系 / 比较"，
> 我可以把它演出来？**

| 内容类型 | 内在动作（值得演） | 实现方向 |
|---|---|---|
| 数字 / 分数 / 排名 | **数字递增**到目标值；多个数字**赛跑**比较 | JS rAF 递增 + 横条 scaleX 同步 |
| 公式 / 差值 / 计算 | 把**减 / 除 / 累加**真的演出来 | 旧字符飞走，新字符落入；箭头连结 |
| 步骤 / 流程 / 流水线 | 节点**依次点亮 + 连线**生长 | SVG path 描边接力 |
| 对比（before / after） | **wipe** 或 **slider drag** 一刀切开 | clip-path inset 推进 |
| 排行榜 / 数据 | 横条**生长**到对应宽度，排序时 swap | width keyframes + FLIP |
| 转换 / 变形 | 形状 A **morph** 到形状 B | SVG path morph / clip-path |
| 出现 / 揭示真相 | **mask reveal** / **聚光灯**扫开 | radial-gradient mask 移动 |
| 文字 / 引用 / 标语 | **typewriter** / **逐字**入场 | width steps / split chars |
| 自绘装饰 | SVG **`stroke-dashoffset`** 描边 | dash 1 → 0（每章至少 1 处） |

**只有**当内容确实没有内在动作（slug / 章节 chip / 标语 hero / 品牌
名 reveal 这种），才退化为**入场动画**（blur clear / fade-up / slide-in）—— 而且**这种 step 全片不能超过 30%**，否则整片就是 PPT。

**装饰性持续微动**（ken burns / breath / pulse / sweep）：

- **不是必填项**。多数 step 不需要。
- 只在该步**一停就会显得过分静止**（例如黑场 hero 文字停留很久）时
  才挂**一层**，且必须低强度（透明度差 ≤ 0.15、scale 差 ≤ 0.015）。
- **整片**至多 30~40% 的 step 有持续微动；不要每步都挂。

**反模式（"AI 味浓"翻车点）**：

- ❌ 每步都给文字加"字符光晕呼吸" / "ken burns" / "accent 横线 sweep"
- ❌ 把"持续微动"当成补足"画面在动"的占位手段
- ❌ 用 blur clear / fade-up 把所有内容入场，不思考内容本身能不能动
- ❌ "三层动作必须齐"的机械套用 —— 这条**已废**

**视觉手段全栈**：CSS filter / SVG stroke 自绘 / clip-path / Canvas
粒子 / mix-blend-mode / typewriter / 数字 ticker / FLIP 排序 / SVG
path morph —— 工具箱见 [`CHAPTER-CRAFT.md`](CHAPTER-CRAFT.md)
Part 2「视觉手段全栈工具箱」。

**字号 / 文字密度（默认上限）**：≤ 1 行 hero（≥ 80px）+ ≤ 1 行次级
（≤ 36px）+ 可选 1 行 mono cue（≤ 14px）。禁整段正文 / 5+ 项小字号
列表。

> **字号同样可被主题覆盖**：`paper-press` / 信息图风格主题可放宽到
> 多行；具体见各主题 `theme.json` 的 `typeScale` 字段。

## 8. 多点内容必须逐个揭示，绝不同时上

脚本里说"三个原因 / 四个特性"时，**1 项 = 1 step**。同步 stagger 是
PPT 直觉。判断标准：**点击之前若已能读到下一个想法 = 揭示太早了**。

| 单步网格 stagger（OK） | 点击驱动逐个揭示（OK） |
|---|---|
| 单元格可互换、无语义 | 每个单元格有名字 / 标签 |
| 传达"很多 / 大量"，扫一眼 | 脚本枚举，每项要读完 |
| ≥ 8 格 | 2-6 项 |

*讲者是否一个一个念出来？* 是 = 逐个揭示。

**布局套路**：N 格网格，`i < step ? <Card /> : <Ghost />`。布局不重排，
只是单元格内容变化。

---

## 9. 整片同一主题

章节间翻转表面色 = 视频里像硬剪辑。想要"暗一点的氛围"段落，在同一调色板
内**降对比、收聚光**，**不要**翻 surface。

**章节 CSS 只用语义 token**（`var(--text)` / `var(--accent)` / ...），
**禁止硬编码 hex / px / 字体名 / duration** —— 否则换主题会失败。

token 速查 + 反例见 [`CHAPTER-CRAFT.md`](CHAPTER-CRAFT.md)；完整契约
见 [`THEMES.md`](THEMES.md)。

## 10. 双源原则：`script.md` 定节拍 / `article.md` 定画面密度

视频是"音 + 画"组合，**画面信息密度必须 > 口播**，否则就是口播稿打印
在屏幕上 = PPT。

| 维度 | 跟谁走 | 为什么 |
|---|---|---|
| step 数 / 估时 / 节拍切分 / hero 标语 | **`script.md`** | 视频是音画时序对齐 |
| hero 内容（口播此刻在念什么） | **`script.md`** | 一节拍 = 一 step = 一聚焦想法 |
| 画面信息（数据 / 引用 / 对比 / 案例细节 / 引用原话） | **`article.md`** | script 为了能口播 3-4 分钟会大量删信息，画面要把这些细节补回来 |
| 装饰元素（角标 / 注释 / mono cue / 信息浮层） | **`article.md`** | "口播没念但屏幕能挂"的事实 |

**核心检查**：每个 step 的画面里，是否有"口播没念出但 article 提到"
的东西？（数字 / 引用 / 对比项 / 标签 / 注释 ...）

- **没有** = 画面 = 口播打印 = 信息密度太薄 = PPT 警报
- **有** = 视频感 = 嘴上 + 画上分别讲不同信息 = 观众两路同时吸收

**反模式 vs 正模式**（抽象示意）：

```
口播：[主体 X] 在 [测评 Y] 排第一。

❌ 反模式：画面写"X 第一"
✓ 正模式：画面同时显示
   - 具体分数（来自 article 第 N 段）
   - 第二名差距 / 领先程度
   - 测评窗口 / 投票数 / 时间戳
   等口播省略了的具体数据
```

> **没 article（用户直接给 script）时**：这一步退化为"主动设计画面
> 信息密度"—— 用数字滚动、对比图、mono cue 元数据等让画面**比口播
> 再多说一些**。**纯口播打字机一定不行**。
